VPC
    -> VPC isolates to an region wise
    -> VPC use CIDR block that defines the IP address (we can use upto 65,536IP's)
    -> Every regions has Default VPC with default Subnets, Secrutiy Group and NACL
    -> The default VPC and its subnets has outbound access to the internet
        -> means out ec2 instance can connect to install for their work ex: npm install,download some tools files for our ec2 instance
    -> Default VPC creates one subnet per Avaialbilty zones
        -> Because if one AZ fails it will work on other AZ.
        Ex: In Mumbai region we have 3 AZ's (ap-south-1a, ap-south-1b, ap-south-1c)
        So for each AZ the default VPC will create one subnet. 
    -> Secrutiy Group in default VPC has default outbound
        -> Means our Ec2 can access out internet.
    -> NACL in default VPC its fully open for inbound and outbound.

So the two layers look like this:
Internet
   ↓
[ NACL — gate is open both ways ✅ ]
   ↓
[ Security Group — lets out freely, blocks entry ✅ ]
   ↓
EC2 instance
